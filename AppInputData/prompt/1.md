# Prompt cho Antigravity: xuất JSON từ Excel `Master_Events`

```text
Tôi sẽ upload 1 file Excel cho bạn. Hãy làm đúng theo yêu cầu dưới đây, ưu tiên sửa/tận dụng app hiện có tên `AppInputData`, không làm lại từ đầu nếu không cần.

## Mục tiêu
Dùng file Excel đã upload, đọc **sheet `Master_Events`** và cho phép app `AppInputData` tự trích xuất dữ liệu thành các file JSON event.

## Phạm vi bắt buộc
- Chỉ dùng **sheet `Master_Events`** làm nguồn dữ liệu.
- Không dùng sheet khác để xuất chính.
- Không đổi cấu trúc dữ liệu gốc nếu không cần.
- Không refactor rộng toàn app nếu không cần.
- Ưu tiên thêm đúng chức năng import Excel -> parse sheet -> export JSON.

## Kết quả đầu ra mong muốn
Khi chạy chức năng export từ `AppInputData`, app phải tạo ra 3 thư mục output:
- `json10`
- `json11`
- `json12`

Mỗi event phải được xuất thành **1 file JSON riêng**.

### Quy tắc đặt file
- Tên file JSON = giá trị cột `id`
- Ví dụ: nếu `id = cach-mang-thang-tam-1945` thì file là:
  - `cach-mang-thang-tam-1945.json`

## Quy tắc phân thư mục theo lớp
Dựa vào field `coverage.grades` của từng row.

Yêu cầu xử lý:
- Nếu event thuộc lớp 10 -> xuất vào `json10`
- Nếu event thuộc lớp 11 -> xuất vào `json11`
- Nếu event thuộc lớp 12 -> xuất vào `json12`
- Nếu `coverage.grades` chứa nhiều lớp, thì **ghi file đó vào tất cả thư mục tương ứng**
  - ví dụ: grades = [10,11] -> cùng 1 JSON được ghi vào cả `json10` và `json11`
- Nếu thiếu `coverage.grades` hoặc parse không được -> đưa vào danh sách lỗi/skip và báo rõ

## Yêu cầu đọc dữ liệu từ Excel
1. Mở workbook Excel tôi upload.
2. Chỉ đọc sheet `Master_Events`.
3. Dùng dòng header hiện có làm tên cột.
4. Bỏ qua các row trống hoàn toàn.
5. Bỏ qua row không có `id`.
6. Trim khoảng trắng đầu/cuối cho dữ liệu text.
7. Chuẩn hóa các giá trị rỗng:
   - ô trống / null / undefined / NaN -> coi là thiếu dữ liệu
8. Nếu app đang có import JSON rồi, hãy thêm import Excel chứ không phá flow cũ.

## Mapping dữ liệu
Sheet `Master_Events` đang ở dạng cột phẳng. Hãy map về JSON event lồng nhau.

Ví dụ:
- cột `titles.primary` -> JSON `{ "titles": { "primary": ... } }`
- cột `classification.eventType` -> JSON `{ "classification": { "eventType": ... } }`
- cột `textbookContent.canonicalSummary` -> JSON tương ứng
- cột `mapData.displayGeometry.marker.lat` -> JSON lồng đúng cấp

### Quy tắc map
- Cột dạng `a.b.c` -> map thành object lồng nhau.
- Cột dạng mảng nếu đang lưu kiểu text -> parse hợp lý.
- Chỉ tạo field khi có dữ liệu hợp lệ.
- Không tạo quá nhiều field rỗng trong JSON output.

## Parse mảng / object
Nếu trong sheet có các field như:
- `coverage.grades`
- `classification.tags`
- `textbookContent.keyFacts`
- `hierarchy.childIds`
- `associations.relatedEventIds`
- `mapData.displayGeometry.provinceNames`
- `mapData.displayGeometry.historicalLocations`
- `media.items...`

thì hãy parse theo quy tắc an toàn sau:

1. Nếu cell đã là JSON string hợp lệ (`[ ... ]` hoặc `{ ... }`) -> parse JSON.
2. Nếu cell là chuỗi phân tách bằng dấu `|` hoặc `,` -> tách thành mảng sau khi trim.
3. Nếu cell chỉ có 1 giá trị -> vẫn cho vào mảng nếu field đó là array field.
4. Nếu parse lỗi -> giữ nguyên string và ghi cảnh báo vào log.

## Kiểu dữ liệu
Hãy cố gắng parse đúng kiểu:
- number cho year, lat, lng, pageStart, pageEnd, level, orderInParent...
- boolean cho các field kiểu true/false
- array cho các field danh sách
- object cho field lồng
- string cho text

Không được để toàn bộ dữ liệu thành string nếu có thể parse đúng kiểu.

## Chuẩn JSON output
Mỗi file JSON phải là 1 event object hoàn chỉnh, có thể dùng ngay cho backend/frontend sau này.

Yêu cầu:
- pretty print, indent 2 spaces
- UTF-8
- giữ tiếng Việt đúng dấu
- không chèn comment vào file JSON

## Validation tối thiểu trước khi xuất
Mỗi row trước khi xuất cần check:
- có `id`
- có `titles.primary` hoặc tên chính tương đương
- có `coverage.grades`

Nếu thiếu các trường tối thiểu này:
- không export file đó
- đưa vào danh sách lỗi

## Tích hợp vào app `AppInputData`
Ưu tiên làm ngay trong app hiện có:
- thêm chức năng chọn file Excel
- đọc sheet `Master_Events`
- preview số row hợp lệ / số row lỗi
- nút `Export JSON`
- sau khi export xong, hiển thị:
  - số file đã tạo
  - số row bị skip
  - danh sách lỗi chính

Nếu app hiện tại đã có nút export, hãy nối logic mới vào đó thay vì tạo luồng song song rối hơn.

## Output structure mong muốn
Ví dụ output:
- `/output/json10/...`
- `/output/json11/...`
- `/output/json12/...`

hoặc cấu trúc tương đương trong project hiện tại, miễn là có đúng 3 folder tách theo lớp như trên.

## Log / báo cáo sau khi chạy
Sau khi hoàn thành, hãy báo cáo rõ:
1. Đã đọc được bao nhiêu row từ `Master_Events`
2. Có bao nhiêu row hợp lệ
3. Có bao nhiêu row bị skip
4. Đã export bao nhiêu file vào `json10`
5. Đã export bao nhiêu file vào `json11`
6. Đã export bao nhiêu file vào `json12`
7. Những cột nào phải parse đặc biệt
8. Những lỗi dữ liệu nào còn tồn tại

## Ưu tiên thực hiện
Thứ tự ưu tiên:
1. Import được Excel
2. Đọc đúng sheet `Master_Events`
3. Map đúng từ cột phẳng -> JSON lồng nhau
4. Tách đúng file theo `coverage.grades`
5. Tên file = `id`
6. Tích hợp vào `AppInputData`
7. Có preview + log lỗi

- Không đổi schema JSON hiện có
- Không dùng dữ liệu ngoài file Excel này để export

## Khi hoàn tất
Hãy cho tôi biết:
- file/code nào đã sửa
- cách chạy chức năng export
- output nằm ở đâu
- ví dụ 1 event JSON đã xuất thành công
```
